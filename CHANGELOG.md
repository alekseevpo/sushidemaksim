# Журнал разработки (Changelog)

## [2026-03-07] - SEO, UX, Compliance & Mobile-First Adaptation

### 📱 Mobile-First Adaptation (Phase 1)
- **Sticky Horizontal Categories**: Реализована «липкая» прокручиваемая лента категорий для мобильных устройств. Теперь пользователям не нужно скроллить наверх, чтобы сменить раздел меню.
- **Thumb-friendly UI**: Кнопки категорий оптимизированы для нажатия большим пальцем, добавлена поддержка горизонтального скролла с плавным затуханием.
- **Header-Category Synergy**: Исправлено перекрытие хедера и панели категорий при прокрутке.

### 🔍 SEO & Поиск
- **JSON-LD Schema**: Реализована микроразметка `FoodEstablishment` на главной и динамическая разметка `Menu/MenuItem` в разделе меню. Теперь Google будет отображать цены и рейтинги прямо в поиске.
- **SEO Component**: Улучшен компонент SEO — добавлена поддержка JSON-LD и оптимизированы мета-теги для всех страниц.

### 🛒 Корзина и Продажи
- **Free Delivery Progress**: Добавлен индикатор прогресса «До бесплатной доставки осталось...» (порог 25€).
- **Upselling (Допродажи)**: Реализован блок «Complementos populares» с рекомендациями соусов и десертов перед оформлением заказа.
- **Repeat Order**: В историю заказов добавлена кнопка «Repetir Pedido», позволяющая в один клик перенести товары из прошлого заказа в текущую корзину.

### 🍱 UI/UX и Контент
- **Premium Placeholders**: Создана система заглушек для товаров без фото. Интегрировано брендированное изображение с надписью "FOTO TEMPORALMENTE NO DISPONIBLE".
- **GDPR Cookie Consent**: Добавлен анимированный баннер согласия на куки с современным дизайном (glassmorphism).
- **Навигация**: Исправлены ссылки «Volver a la tienda» — теперь они ведут в меню товаров, а не на главную, сокращая путь пользователя к покупке.
- **Фикс типов**: Устранены ошибки в компоненте корзины при добавлении рекомендованных товаров.

### 📈 Маркетинг и Конверсия (Social Proof & FOMO)
- **FOMO Badges**: Добавлены премиальные бейджи "Top Ventas", "Chef Choice" и "Nuevo" для увеличения доверия и авторитетности блюд.
- **Динамические счетчики**: Внедрен индикатор реального спроса "Pedido X veces hoy" с анимацией пульса для создания эффекта востребованности.
- **Smart Empty Cart**: Реализован блок "Nuestros favoritos hoy" в пустой корзине, позволяющий пользователю начать покупки в один клик.
- **Mobile Floating Cart**: Добавлена плавающая и липкая кнопка корзины для мобильных устройств, обеспечивающая быстрый доступ к оформлению заказа.
- **Allergen System**: Интегрированы иконки аллергенов (Gluten, Lácteo, Pescado и др.) с возможностью управления ими через админ-панель.

### 🔐 Безопасность (Security Hardening)
- **User Enumeration Protection**: Исправлена утечка данных в форме забытого пароля. API больше не подтверждает наличие email в базе, защищая приватность клиентов.
- **Order Logic Protection**: Закрыта уязвимость, позволявшая пользователям самостоятельно менять статусы доставки своих заказов.
- **Code Hardening**: Удалены все тестовые промокоды, зашитые в код, исключая возможность несанкционированного получения скидок.
- **Data Integrity**: Улучшена валидация данных на бэкенде при добавлении маркетинговых меток и аллергенов.


## [2026-03-06] - Миграция в облако и деплой

### 🚀 Деплой

- **Vercel**: Проект успешно задеплоен как единое целое (Frontend + API).
- **Vercel Cron**: Настроена задача для проверки опоздавших заказов. Расписание оптимизировано для Hobby-плана (1 раз в сутки).

### 🛠 Бэкенд и База данных

- **Миграция на Supabase**: Полный переход с локальной SQLite на PostgreSQL в облаке Supabase.
- **Миграция данных**: Перенесено **57 блюд** из локальной базы в облако.
- **Безопасность**: Настроены Row Level Security (RLS) политики в Supabase для безопасного доступа к данным.
- **Фикс типов**: Исправлены ошибки приведения типов для PostgreSQL (ID как Integer, Boolean значения для чекбоксов).

### 🖼 Хранилище (Storage)

- **Supabase Storage**: Настроена загрузка изображений в облачный бакет `images` вместо локальной файловой системы (необходимое условие для работы на Vercel).
- **Превью изображений**: Исправлена загрузка и отображение миниатюр в админ-панели и основном меню.

### 👤 Администрирование

- **Role Management**: Настроены права Супер-администратора для управления контентом и ролями.
- **Admin UI**: Исправлены ошибки при сохранении и редактировании блюд (поля Picante, Vegetariano, Promoción).

### 🐛 Исправления ошибок

- Исправлена ошибка `editAddress` в `ProfilePage.tsx`, блокировавшая сборку.
- Оптимизированы переменные окружения для работы в серверлесс-среде.
