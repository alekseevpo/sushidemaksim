# 🚀 Инструкция по переезду на sushidemaksim.com

Чтобы процесс перехода прошел бесшовно и ссылки в письмах обновились автоматически, следуйте этой инструкции.

## 1. Настройка переменных окружения (Env Vars)

Вам **не нужно** менять код. Вместо этого настройте переменную на вашем хостинге (например, Vercel):

1. Зайдите в **Project Settings**.
2. Перейдите в раздел **Environment Variables**.
3. Добавьте новую переменную или измените существующую:
   - **Name**: `FRONTEND_URL`
   - **Value**: `https://sushidemaksim.com` (без косой черты в конце)
4. Сохраните и сделайте **Redeploy** (пересборку) проекта.

## 2. Как это работает в коде

Я уже внедрил в файл `server/src/config.ts` логику приоритетов:

```typescript
frontendUrl: process.env.FRONTEND_URL || (nodeEnv === 'production' ? 'https://sushidemaksim.vercel.app' : 'http://localhost:5173')
```

1. **Приоритет 1**: Если задана переменная `FRONTEND_URL` (например, `.com`), используется она.
2. **Приоритет 2**: Если переменной нет, используется `sushidemaksim.vercel.app` (ваш текущий рабочий сайт).
3. **Приоритет 3**: Если вы запускаете сайт локально для тестов, используется `localhost:5173`.

## 3. Чек-лист при переезде

- [ ] Привязать домен `sushidemaksim.com` в панели Vercel (Domains).
- [ ] Обновить `FRONTEND_URL` в Environment Variables.
- [ ] Проверить настройки CORS (я уже добавил `.com` в разрешенные списки в `config.ts`, так что запросы будут проходить).
- [ ] Отправить тестовое письмо лояльности и нажать «PEDIR AHORA».

> [!TIP]
> Использование переменных окружения — это самый надежный способ. Это позволяет одному и тому же коду работать на разных адресах (тестовом и рабочем) без правок.
