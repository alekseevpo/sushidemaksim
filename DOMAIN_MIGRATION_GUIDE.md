# 🚀 Инструкция по переезду на sushidemaksim.com

Чтобы процесс перехода прошел бесшовно, мы внедрили динамическую конфигурацию домена. Следуйте этой инструкции завтра.

## 1. Настройка переменных окружения (Env Vars)

Вам нужно настроить переменные в панели управления Vercel:

1. Зайдите в **Project Settings** -> **Environment Variables**.
2. Добавьте/обновите следующие переменные:
    - **Frontend (Client)**:
        - **Name**: `VITE_SITE_URL`
        - **Value**: `https://sushidemaksim.com` (без / в конце)
    - **Backend (Server)**:
        - **Name**: `FRONTEND_URL`
        - **Value**: `https://sushidemaksim.com`
3. Сохраните и сделайте **Redeploy** (пересборку) проекта.

## 2. Как это работает в коде

Я внедрил централизованную систему подхвата домена:
- **Фронтенд**: Использует константу `SITE_URL` из `src/constants/config.ts`.
- **Бэкенд**: Использует конфиг из `server/src/config.ts`.

Если переменные не заданы, система автоматически использует `https://sushidemaksim.com` как значение по умолчанию.

## 3. Чек-лист в день переезда (Завтра)

- [ ] В панели Vercel привязать домен `sushidemaksim.com` (Раздел Domains).
- [ ] Назначить его основным (**Primary Domain**).
- [ ] Настроить старый адрес `sushidemaksim.vercel.app` как **Redirect** на новый домен.
- [ ] Обновить переменные окружения (пункт 1).
- [ ] Запустить `generate-sitemap.js` (через CI/CD или вручную), чтобы обновить файл `sitemap.xml`.
- [ ] В Google Search Console использовать инструмент **"Change of Address"** (Изменение адреса), указав новый домен.

> [!TIP]
> Использование `VITE_SITE_URL` гарантирует, что все SEO-теги, канонические ссылки и микроразметка обновятся одновременно с доменом. 🛰️📈
